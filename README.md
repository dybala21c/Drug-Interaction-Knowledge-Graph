# Drug Interaction Knowledge Graph

## Overview

This project focuses on the construction and enrichment of a biomedical Knowledge Graph for representing drug-related knowledge.

The main objective is to model relationships between drugs, active ingredients, and side effects, and to explore the use of Large Language Models (LLMs) for Knowledge Graph enrichment.

## Research Motivation

Biomedical information is complex, heterogeneous, and continuously evolving. Knowledge Graphs provide a structured representation of biomedical entities and relationships, enabling reasoning and intelligent information retrieval.

This project explores the combination of:

- Knowledge Graph technologies
- Graph databases
- Large Language Models
- Artificial Intelligence approaches

for biomedical knowledge representation and enrichment.

## Knowledge Graph Schema

The current Knowledge Graph contains three main types of entities:

### Entities

- **Drug (Medicament)**
- **Active Ingredient (IngredientActif)**
- **Side Effect (EffetSecondaire)**

### Relations

- `INTERACTS_WITH`
- `CAN_CAUSE`
- `HAS_ACTIVE_INGREDIENT`

## Technologies

The project is implemented using:

- Python
- Neo4j Graph Database
- Cypher Query Language
- Large Language Models (LLMs)
- Graph-based reasoning approaches

## Main Features

- Construction of a biomedical Knowledge Graph
- Representation of drug interactions
- Modeling of biomedical entities and relations
- Graph visualization
- Exploration of LLM-based Knowledge Graph enrichment

## Future Research Directions

This project is a foundation for exploring:

- Knowledge Graph Embeddings
- Explainable AI for Knowledge Graph reasoning
- Neuro-Symbolic approaches
- Retrieval-Augmented Generation (RAG) with Knowledge Graphs
